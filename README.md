# AI Agent CLI Tool (Website Cloner)

A conversational CLI agent designed to autonomously clone modern websites (e.g., Scaler Academy) using HTML, CSS, and JavaScript. Built on Node.js using the OpenAI SDK configured for the Gemini API.

## Core Features
- **Conversational CLI**: Interact with the agent directly from your terminal using natural language.
- **Strict JSON Loop**: The agent reasons through tasks using a rigid `START | THINK | TOOL | OBSERVE | OUTPUT` workflow.
- **Live Web Fetching**: Employs an `axios`-based `fetchWebsite` tool to dynamically scrape live URL content, enabling the agent to analyze actual HTML/CSS structure before writing its clone.
- **Secure File Sandboxing**: All files generated by the LLM (using the `writeFile` tool) are safely intercepted and forcefully placed inside a dedicated `cloned-website/` directory. This neutralizes directory traversal vulnerabilities and prevents the LLM from overwriting critical project files.
- **Robust Model Fallback**: If the primary preview endpoint (`gemini-3.1-flash-lite-preview`) hits its 15 RPM free-tier rate limit or experiences downtime, the agent automatically catches the `429/404` errors and seamlessly cycles down to stable fallback models (`gemini-2.5-flash`, `gemini-2.0-flash`, `gemini-1.5-flash`) without breaking the active chat session.

## Prerequisites
- Node.js installed
- A free Gemini API key from [Google AI Studio](https://aistudio.google.com/)

## Setup

1. Install dependencies:
   ```bash
   npm install
   ```

2. Create a `.env` file in the root directory and add your API key:
   ```
   GEMINI_API_KEY=your_api_key_here
   ```

3. Run the CLI tool:
   ```bash
   node index.js
   ```

## Usage
Once the CLI starts, you can type natural language instructions. To run the assignment task, provide the following prompt:
> "Clone the Scaler Academy website with a Header, Hero Section, and Footer. First, use your fetchWebsite tool to read https://www.scaler.com so you can accurately clone its structure and style."

The agent will output its reasoning steps in the terminal and automatically generate the necessary HTML/CSS files directly into the `cloned-website/` folder.
